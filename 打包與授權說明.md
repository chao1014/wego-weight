# 過磅系統 (Weight) - 打包與授權管理說明書 (管理端)

本說明書旨在指導過磅系統管理員/開發人員如何對過磅系統進行 **一鍵安全加固打包** 以及如何產生與管理 **設備授權金鑰**。

---

## 📂 打包與授權檔案結構

所有打包配置與授權的核心元件均統一收納於：  
👉 **[packaging_tools](file:///D:/taekwondo-suite/weight/packaging_tools/)** 資料夾

*   **`build_all.py`**：自動化加固打包腳本。整合 PyArmor 程式碼加密與 PyInstaller，將過磅系統以資料夾模式（onedir）打包，並內建數據保護備份機制。
*   **`license_generator.py`**：管理員專用 GUI，用來輸入機器碼並產生 `license.lic`。
*   **`license_verifier.py`**：內嵌於過磅主程式中的授權驗證函式庫。
*   **`register_gui.py`**：使用者端的授權錯誤提示與「一鍵匯入」註冊視窗。

---

## 🛠️ 步驟一：如何進行一鍵加固打包

當您修改了過磅系統程式碼，需要產出最新版本的免安裝 `.exe` 檔給使用者：

1. 開啟終端機（如 PowerShell 或 cmd），進入本系統根目錄：
   ```bash
   cd D:\taekwondo-suite\weight
   ```
2. 執行打包指令：
   ```bash
   python packaging_tools/build_all.py
   ```
3. **打包成果**：
   打包完成後，免安裝發布資料夾會輸出至 **`weight/dist/`**。
   裡面會包含編譯完成的 **`WeighIn`** 免安裝程式資料夾。
   *   `dist/WeighIn/WeighIn.exe` (過磅系統主程式)
   *   `dist/WeighIn/_internal/` (PyInstaller 編譯之相依檔案與執行庫資料夾)
   *   `dist/WeighIn/templates/` & `static/` (網頁後端資源)
   *   `dist/WeighIn/labels/` (過磅標籤範本)
   *   `dist/WeighIn/photos/` (相片儲存資料夾)
   *   `dist/WeighIn/config.json` (設定檔)

> [!NOTE]
> *   **單一授權部署**：請將產生的 `license.lic` 授權檔案直接放置於 `dist/WeighIn/` 目錄下（與 `WeighIn.exe` 放在一起），程式即可正常偵測與載入。
> *   **資料庫初始化**：程式在啟動時會自動偵測 `weigh_in.db` 資料庫檔案，若不存在會自動建立。
> *   **重新打包數據保護機制**：為了解決重新打包清理舊目錄時，會導致已部署的授權與過磅歷史數據被誤刪的問題，打包腳本已整合自動備份與還原機制。重新打包時，`license.lic`、`.sys_time.dat`、過磅歷史資料庫 **`weigh_in.db`** 與選手相片目錄 **`photos/`** 將會**被自動備份並在打包後還原**，確保過磅數據 100% 不會丟失。

## 🔑 步驟二：如何為新設備產生授權

本系統的授權綁定主機硬體（主機板 UUID 與 CPU ID）。每一台電腦在執行程式時，必須在 `dist/` 根目錄下擁有專屬且簽章正確的 `license.lic` 檔案。

### 📌 取得本機機器碼的兩種方法

> [!TIP]
> **本機設備機器碼的標準格式**：  
> `TK-XXXX-XXXX-XXXX-XXXX` (共 16 碼十六進位，以 `TK-` 開頭，由 4 個破折號分隔之 4 碼區塊組成)。同一台電腦在主系統、過磅系統、公眾端系統算出的機器碼皆會完全相同。

*   **方法一：使用者端雙擊程式自動獲取（最常用）**  
    請使用者在電腦上開啟 `WeighIn.exe` 程式，畫面上將會彈出「設備授權註冊」視窗，請使用者點擊「**複製機器碼**」並將該碼傳送給您。
*   **方法二：開發者端終端機指令快速獲取**  
    在當前電腦的終端機（PowerShell 或 CMD）中執行以下指令，即可直接在螢幕上列印出本機機器碼：
    ```bash
    python packaging_tools/license_verifier.py
    ```

### 📝 產生授權憑證步驟

1. **啟動授權產生器**：
   在開發環境下執行：
   ```bash
   python packaging_tools/license_generator.py
   ```
2. **填寫授權資訊**：
   *   **授權對象**：輸入該設備名稱或體育館名稱（例如：`Official User`）。
   *   **設備機器碼**：貼上取得的 `TK-XXXX-XXXX-XXXX-XXXX` 機器碼。
   *   **到期日期**：格式為 `YYYY-MM-DD`（預設填寫為 `2026-12-31`）。
   *   **授權模組**：勾選 `過磅主系統 (weight)` 或 `萬用全功能授權 (*)`。
3. **生成檔案**：點擊「產生並儲存授權檔案」，將產出的 **`license.lic`** 檔案傳送給使用者，讓使用者直接放置於 `dist/WeighIn/` 資料夾內（與 `WeighIn.exe` 放在一起）即可。

## 👁️ 使用者如何檢視授權期限？

1.  **檔案中檢視**：使用者可以直接以文字編輯器打開 `dist/license.lic`，裡面的 `"expire_date"` 欄位即代表其到期日期。
2.  **主視窗標題列檢視**：當過磅程式啟動時，過磅視窗最上方標題列的右側，會動態顯示 `[已授權至: YYYY-MM-DD]`（例如：`韻動國際過磅系統 [已授權至: 2026-12-31]`）。

## ⚡ 自動更新機制 (Auto-Updater)

本系統支援「免版本號管理」的一鍵自動更新機制。過磅程式啟動時會自動對比伺服器與本地檔案特徵，以在無感狀態下維持最新版。

### 📡 1. 部署更新檔 (管理端)
當您有新編譯好的過磅軟體釋出時：
1. **直接複製**：將最新編譯的 `WeighIn.exe` 檔案複製覆蓋丟入您的共用更新目錄：
   👉 **`Y:\競賽軟體\`** 即可，不需要做其他任何操作。

### ⚙️ 2. 設定更新來源 (用戶端)
*   **更新設定**：在 `dist/WeighIn/config.json` 的 `"update_source"` 欄位設定。
*   **啟用更新**：設定 `"update_source": "Y:\\競賽軟體"` (預設已設定為此值)。
*   **關閉更新**：將 `"update_source"` 欄位留空即可。

### 🔄 3. 更新比對與防鎖定覆寫流程
1. **特徵比對**：程式啟動時，會自動比對本地 `WeighIn.exe` 與 `Y:\競賽軟體` 下執行檔的 **「檔案大小」** 與 **「修改時間」**。
2. **相同秒開**：特徵相同表示已是最新版，程式將在幾毫秒內秒開，不產生任何開機延遲。
3. **不同更新**：若特徵不同，程式會彈出更新提示。使用者同意後，程式會背景下載新版，自動調用 `temp_update/update_helper.bat` 並退出舊程式釋放鎖，隨後由批次檔完成檔案覆寫並重新開啟新版。

## 🛡️ 離線防時光倒流（防竄改）機制

本系統具備離線狀態下的雙重防時間竄改防護，無須連接網際網路即可有效防止使用者將本機電腦時間往回調以繞過授權期限：

1. **簽發日期卡控 (`issue_date`)**：
   * 在使用 `license_generator.py` 產生授權時，系統會自動在 `license.lic` 中記錄當天的 `issue_date` 並將其納入 HMAC 簽章。
   * 程式啟動時，若本機時間早於簽發日期，將會被判定為時間異常而拒絕啟動。（向下相容：若為不含此欄位的舊授權，則預設以 `2026-01-01` 作為限制）。
2. **最後執行時間追蹤 (`.sys_time.dat`)**：
   * 授權驗證通過後，程式會在 `license.lic` 同目錄（即打包後的 `dist/` 根目錄）下自動產生一個加密且隱藏的時間紀錄檔 `.sys_time.dat`。
   * 每次正常啟動時，系統會驗證此檔案的簽章，並檢查本機目前時間是否早於該檔記錄的「最後執行時間」：
     * 若本機時間被調回過去，將會攔截並報錯 `TIME_TRAVEL_BACKWARD`。
     * 若手動用記事本修改該檔案以試圖推前時間，會因 HMAC 簽章不符而報錯 `TIME_RECORD_TAMPERED`。
     * 驗證通過後，系統會自動將該檔內容更新為目前最新本機時間。
